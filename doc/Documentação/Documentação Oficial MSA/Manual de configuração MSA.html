Configuração do MSA

Keycloak

O Keycloak é uma solução open-source de autenticação e autorização criada pela Red Hat que funciona como um servidor central para controlar logins de usuários e acessos em aplicações e APIs. No MSA estou usando a versão 26.3.3.

Abaixo será descrito todo preparo para configurar o Keycloak para usar no MSA:

Criando Realm de Teste

No Keycloak temos os realms que são ambientes do Keycloak, como um ambiente de desenvolvimento por exemplo, e nesse caso temos que criar um realm novo, pode ser usando na master, porém vamos criar um realm de teste.

No console do Keycloak temos a aba Manage Realms, nela podemos criar novos realms.

Clique em create.

Voce pode adicionar um JSON para criar um realm predefinido, como o JSON abaixo anexado, contendo o ambiente dev com client MSA ja existentes com todos as roles. Porém será necessário gerar um novo client secret no client MSA e Users no realm dev com as roles do MSA:

Caso queira criar o ambiente passo a passo. Digite o nome do seu realm, no nosso caso teste, e depois clique no create.

Criando Client do MSA

Depois de criar o realm, agora podemos criar o Client. No Keycloak as aplicações que usamos para ter a autenticação do mesmo, são denominado Clients, as nossas aplicações são Clients. Agora vamos criar um Client chamado MSA, que é o cliente da nosso realm.

Vamos criar o ID do client (que é unico por realm), nome e descrição, usando o Client Type OpenID Connect.

Valid redirect URIs: <DNS>:<PORTA>/login/oauth2/code/keycloak

Valid post logout redirect URIs: Rota da aplicação sem / no final

Web Origins: Rota da aplicação com / no final

Após isso, utilize a imagem como configuração.

Criar Roles no Client

Configurar o Mapper para Injetar Tokens Corretos no Spring

Vamos criar as roles, para determinar quem é ADMIN no sistema. No MSA é bem simples, só quem é ADMIN pode acessar a rota /admin/**, ou seja é a role necessária para caracterizar o usuário admin ou não.

Na aba Roles, vamos criar uma roles , que é obrigatória para rodar o MSA.

Crie todas apresentadas abaixo:

Roles de rotas:

Nome da RoleDescrição
index_adminAcesso a rota home do portal admin
search_adminAcesso a rota search do portal admin
create_adminAcesso a rota create do portal admin

Roles de permissão:

Nome da RoleDescrição
ADMIN_CREATERole de permisão para criar hosts
ADMIN_READRole de permisão para ler hosts
ADMIN_UPDATERole de permisão para editar hosts
ADMIN_DELETERole de permisão para excluir hosts

Vamos clicar no MSA-dedicated.

Configurar uma nova regra, um novo mapper.

Role para baixo e encontrará o mapper User Client Role, clique nele.

Utilize as mesmas configurações abaixo, que configuração o Client MSA a ter um mapeamento direto para o Client usando as roles da própria aplicação, que fica no resource_access.MSA.roles.

Criando Usuários para logar no MSA

Agora vamos criar um usuário com permissão de admin. Na aba Users podemos criar um novo usuário para o MSA.

Escolha um username, que é único por Realm, email, primeiro e último nome.

Configure a senha e desative o Temporary, para ter uma senha fixa.

Na aba de Role Mapping vamos adicionar a Client Roles , criada no cliente MSA, para o usuário criado.

Depois disso o Keycloak está configurado com sucesso, agora partimos para a aplicação MSA, o Spring Boot.

Banco de Dados

No caso do banco de dados, o MSA usa o MySQL, para funcionar o MSA é necessário criar a table msa para que o MSA persista os dados. A imagem abaixo mostra a tela inicial na conexão de um servidor MySQL usando o CLI Client.

Podemos criar um table chamada msa no MySQL com o comando abaixo:

create database msa;

Rodando o MSA

Acessando a rota localhost:8080/admin (Mas pode ser diferente para você, caso a porta seja diferente), irá pedir login no Realm setado.
